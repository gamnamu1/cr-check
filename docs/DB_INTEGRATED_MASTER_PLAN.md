## 📋 개요

이 문서는 CR-Check의 데이터베이스 구축을 위한 **단일 진실 공급원(Single Source of Truth)**입니다.
기존 기획 문서의 **전략적 가치(개인화, 통계)**와 실행 계획서의 **기술적 구현(Supabase, Code)**을 통합하고, 누락된 **사용자 프로필 관리** 로직을 보완했습니다.

### 🛠 기술 스택 선정
- **Database**: Supabase (PostgreSQL 15+)
- **Auth**: Supabase Auth (Email, Google, Kakao)
- **Backend**: FastAPI + Supabase Python Client (Service Role)
- **Frontend**: Next.js 15 + Supabase JS Client (App Router ready)

## 💻 상세 구현 명세

### 1. 데이터베이스 스키마 (SQL)

Supabase SQL Editor에서 실행할 최종 스크립트입니다. `profiles` 테이블 트리거가 포함되어 있어 회원가입 시 자동으로 프로필이 생성됩니다.

```sql
-- Transaction 시작
BEGIN;

-- 1. Users Profile (사용자 정보 확장)
CREATE TABLE public.profiles (
  id UUID REFERENCES auth.users(id) ON DELETE CASCADE PRIMARY KEY,
  email TEXT,
  nickname TEXT,
  role TEXT DEFAULT 'user' CHECK (role IN ('user', 'admin')),
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- RLS: 프로필
ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public profiles are viewable by everyone." ON public.profiles FOR SELECT USING (true);
CREATE POLICY "Users can insert their own profile." ON public.profiles FOR INSERT WITH CHECK (auth.uid() = id);
CREATE POLICY "Users can update own profile." ON public.profiles FOR UPDATE USING (auth.uid() = id);

-- Trigger: 회원가입 시 자동으로 public.profiles에 행 생성
CREATE OR REPLACE FUNCTION public.handle_new_user()
RETURNS TRIGGER AS $$
BEGIN
  INSERT INTO public.profiles (id, email, nickname)
  VALUES (new.id, new.email, new.raw_user_meta_data->>'full_name');
  RETURN new;
END;
$$ LANGUAGE plpgsql SECURITY DEFINER;

CREATE TRIGGER on_auth_user_created
  AFTER INSERT ON auth.users
  FOR EACH ROW EXECUTE PROCEDURE public.handle_new_user();


-- 2. Articles (기사 원본 메타데이터)
CREATE TABLE public.articles (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  url TEXT NOT NULL,
  title TEXT NOT NULL,
  publisher TEXT,
  journalist TEXT,
  publish_date TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL,
  
  CONSTRAINT articles_url_key UNIQUE (url)
);

-- Index for Articles
CREATE INDEX idx_articles_url ON public.articles(url);
CREATE INDEX idx_articles_publisher ON public.articles(publisher);

-- RLS: Articles
ALTER TABLE public.articles ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Articles are viewable by everyone." ON public.articles FOR SELECT USING (true);
-- 쓰기는 Service Role(백엔드)만 가능하므로 별도 정책 불필요 (기본 차단)


-- 3. Analysis Results (분석 결과)
CREATE TABLE public.analysis_results (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  article_id BIGINT REFERENCES public.articles(id) ON DELETE CASCADE NOT NULL,
  user_id UUID REFERENCES public.profiles(id) ON DELETE SET NULL, -- 비로그인 가능
  
  -- 리포트 내용
  comprehensive_report TEXT,
  journalist_report TEXT,
  student_report TEXT,
  
  -- 메타데이터
  model_version TEXT DEFAULT 'claude-sonnet-4.5',
  duration_seconds FLOAT,
  detected_categories JSONB, -- Phase 1 결과 (배열)
  
  created_at TIMESTAMP WITH TIME ZONE DEFAULT timezone('utc'::text, now()) NOT NULL
);

-- Index for Analysis
CREATE INDEX idx_analysis_article ON public.analysis_results(article_id);
CREATE INDEX idx_analysis_user ON public.analysis_results(user_id);
CREATE INDEX idx_analysis_created_at ON public.analysis_results(created_at DESC);

-- RLS: Analysis Results
ALTER TABLE public.analysis_results ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Analysis results are viewable by everyone." ON public.analysis_results FOR SELECT USING (true);
-- 쓰기는 Service Role(백엔드)만 가능

COMMIT;
```

#### `database.py` (핵심 로직 개선안)
Service Role Key를 사용하여 RLS를 우회, 쓰기 권한을 확보합니다.

```python
import os
from supabase import create_client, Client
from typing import Optional, Dict, Any

class DatabaseManager:
    def __init__(self):
        url = os.environ.get("SUPABASE_URL")
        key = os.environ.get("SUPABASE_SERVICE_ROLE_KEY")
        if not url or not key:
            print("⚠️ Supabase Credentials Missing. Database disabled.")
            self.client = None
        else:
            self.client: Client = create_client(url, key)

    async def save_or_get_article(self, article_data: Dict) -> Optional[int]:
        """기사 중복 확인 후 ID 반환"""
        if not self.client: return None
        
        try:
            # 1. Check ID
            res = self.client.table("articles").select("id").eq("url", article_data["url"]).execute()
            if res.data:
                return res.data[0]["id"]
            
            # 2. Insert New
            new_article = {
                "url": article_data["url"],
                "title": article_data["title"],
                "publisher": article_data.get("publisher"),
                "journalist": article_data.get("journalist"),
                "publish_date": article_data.get("publish_date") # ISO Format string
            }
            res = self.client.table("articles").insert(new_article).execute()
            return res.data[0]["id"]
        except Exception as e:
            print(f"❌ DB Article Error: {e}")
            return None

    async def save_analysis(self, article_id: int, result: Dict, duration: float, user_id: str = None):
        """분석 결과 저장"""
        if not self.client or not article_id: return
        
        try:
            data = {
                "article_id": article_id,
                "user_id": user_id, # Optional
                "comprehensive_report": result["reports"].get("comprehensive"),
                "journalist_report": result["reports"].get("journalist"),
                "student_report": result["reports"].get("student"),
                "detected_categories": result.get("detected_categories"),
                "duration_seconds": duration,
            }
            self.client.table("analysis_results").insert(data).execute()
            print("✅ Analysis Saved to DB")
        except Exception as e:
            print(f"❌ DB Analysis Error: {e}")
```
