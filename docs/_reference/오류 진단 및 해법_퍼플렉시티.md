## 진단 결과: 두 가지 독립 버그

### 버그 \#1 — `inference_role` 컬럼 부재 (400 Bad Request)

**에러 메시지**:[^2]

```
Client error '400 Bad Request' for url '...pattern_relations?
select=source_pattern_id,target_pattern_id,inference_role
&relation_type=eq.inferred_by'
```

**원인**: 코드가 `pattern_relations` 테이블에서 `inference_role` 컬럼을 SELECT하고 있는데, 실제 DB 스키마에는 그 컬럼이 없습니다. 마스터 플랜의 `pattern_relations` DDL을 보면  `relation_type`, `description`, `source`, `confidence`, `verified` 컬럼만 있고 **`inference_role`은 없습니다.** 배포 후 코드와 DB 스키마가 불일치한 상태입니다.[^3]

**영향**: 메타 패턴 추론 전체 건너뜀. 하지만 이것 자체가 Phase2 실패의 원인은 **아님** — `추론 건너뜀`으로 처리하고 파이프라인은 계속 진행됩니다.

### 버그 \#2 — Phase2(Sonnet 4.6) `JSONDecodeError` → 3회 재시도 후 전패 (핵심 원인)

**에러 메시지** (두 케이스):[^2]

```
# 케이스 A (4월 7일): 3종 리포트 JSONDecodeError
리포트 생성 시도 1/3 실패: [JSONDecodeError] Expecting ',' delimiter: line 10 column 645
리포트 생성 시도 2/3 실패: [JSONDecodeError] Expecting ',' delimiter: line 10 column 643
리포트 생성 시도 3/3 실패: [JSONDecodeError] Expecting ',' delimiter: line 14 column 278

# 케이스 B (4월 8일 오늘): Anthropic 서버 과부하
리포트 생성 시도 1/3 실패: [OverloadedError] Error code: 529 - Overloaded
리포트 생성 시도 2/3 실패: [OverloadedError] Error code: 529 - Overloaded
리포트 생성 시도 3/3 실패: [OverloadedError] Error code: 529 - Overloaded
```

**Supabase 데이터 확인**: `id=2, 4, 5, 7` 모두 `comprehensive_report = "리포트 생성 중 오류가 발생했습니다."`. `phase2_model = claude-sonnet-4-6`은 기록되어 있는데 실제 리포트는 없음. `article_analysis`와 `overall_assessment`는 정상 저장됨 → **Phase1(Sonnet 4.5)은 성공, Phase2 리포트 생성만 실패**.[^1]

엔트로픽 API 로그에서 `sonnet-4-6` 항목에 INPUT/OUTPUT TOKENS가 `-`로 보이는 것도 이 때문입니다  — 529 OverloadedError는 요청이 서버에 도달했다가 즉시 거부된 것이므로 토큰 카운트가 0으로 표시됩니다.[^4]

**케이스 A의 JSONDecodeError 원인 심층 분석**: `"Expecting ',' delimiter"` 에러는 Sonnet 4.6이 3종 리포트를 JSON으로 생성할 때 **긴 한국어 텍스트 내에 이스케이프되지 않은 특수문자(따옴표, 줄바꿈 등)**가 들어가면서 JSON이 깨지는 현상입니다. 3회 재시도해도 같은 패턴이 반복되는 것은, 동일 프롬프트로 재시도하기 때문입니다.

### Fix \#2 — Phase2 JSON 파싱 강화 (핵심 수정)

**케이스 A (JSONDecodeError)**: `_robust_json_parse` 함수가 Phase2 리포트 생성에는 적용되지 않고 있습니다. Phase1(Solo)에는 3차 정규식 추출까지 있는데 Phase2 리포트 파싱에는 `try/except json.JSONDecodeError`만 있는 구조로 보입니다.

```python
# 현재 구조 (추정)
for attempt in range(3):
    response = call_sonnet46(prompt)
    report = json.loads(response.content)  # 여기서 실패 시 그냥 재시도
    
# 권장 구조
import re, json

def robust_parse_report(raw: str) -> dict:
    # 1차: 직접 파싱
    try:
        return json.loads(raw)
    except json.JSONDecodeError:
        pass
    
    # 2차: ```json 블록 추출
    match = re.search(r'```json\s*([\s\S]*?)\s*```', raw)
    if match:
        try:
            return json.loads(match.group(1))
        except json.JSONDecodeError:
            pass
    
    # 3차: 한국어 텍스트의 이스케이프 문제 정규화
    cleaned = raw.replace('\n', '\\n').replace('\r', '\\r')
    try:
        return json.loads(cleaned)
    except json.JSONDecodeError:
        pass

    # 4차: JSON 모드 강제 재요청 (별도 attempt)
    raise ValueError(f"JSON 파싱 완전 실패: {raw[:200]}")
```

**케이스 B (OverloadedError 529)**: 재시도 간격이 너무 짧습니다. 지수 백오프(exponential backoff)를 적용하세요:

```python
import time

for attempt in range(3):
    try:
        response = call_sonnet46(prompt)
        ...
    except OverloadedError:
        if attempt < 2:
            wait = 2 ** (attempt + 2)  # 4초, 8초
            logger.warning(f"529 Overloaded, {wait}초 후 재시도...")
            time.sleep(wait)
        else:
            raise
```
