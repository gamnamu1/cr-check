# 오류 진단 및 해법_마누스

네, 제공해주신 로그와 DB 데이터, 그리고 문제 상황 설명을 종합적으로 분석했습니다. 결론부터 말씀드리면, **두 가지 독립적인 오류가 동시에 발생**하고 있으며, 이로 인해 Phase 2(리포트 생성) 단계로 넘어가지 못하고 파이프라인이 실패하는 것으로 보입니다.

### 종합 진단: 두 가지 핵심 원인

1. **`JSONDecodeError` (1차 원인):** `Sonnet 4.5`가 생성한 1단계 분석 결과(JSON)에 문법 오류가 있어 파싱에 실패하고 있습니다. 이로 인해 후속 단계로 넘어갈 유효한 `detected_patterns` 데이터를 얻지 못합니다.
2. **`OverloadedError` (2차 원인):** 1번 문제가 발생한 후, 재시도 과정에서 `Sonnet 4.6`을 호출할 때 Anthropic API가 과부하 상태(Overloaded)여서 요청이 거부되고 있습니다.

이 두 문제가 겹치면서, `Sonnet 4.5`는 호출되었지만 `Sonnet 4.6`은 토큰 사용량 없이(-) 로그에만 남고 실제로는 실행되지 않는 현상이 나타나는 것입니다.

---

### 상세 분석 및 근거

### 원인 1: `JSONDecodeError` - Sonnet 4.5의 불안정한 JSON 출력

- **로그 증거**: Railway 로그를 보면 명확한 `JSONDecodeError`가 여러 차례 기록되어 있습니다.
    
    > `리포트 생성 시도 1/3 실패: [JSONDecodeError] Expecting ',' delimiter: line 10 column 645 (char 968)`
    > 
- **DB 데이터 증거**: Supabase DB에 저장된 `analysis_results` 테이블의 `article_analysis` 컬럼을 보면, 정상적인 리포트 대신 `(나)가 압도적으로 강하다...` 와 같은 **1단계 분석 결과(Devil's Advocate CoT)의 일부가 그대로 저장**되어 있습니다. 이는 `Sonnet 4.5`가 JSON 형식뿐만 아니라 부가적인 텍스트를 함께 출력했고, `_robust_json_parse`가 이를 파싱하려다 실패했음을 의미합니다.
- **결론**: `pattern_matcher.py`의 `Sonnet 4.5`가 프롬프트 지시를 100% 따르지 않고, JSON 외의 텍스트를 출력하는 것이 근본적인 문제입니다.

### 원인 2: `OverloadedError` - Anthropic API 과부하

- **로그 증거**: Railway 로그에 `OverloadedError`가 명확히 기록되어 있습니다.
    
    > `리포트 생성 시도 1/3 실패: [OverloadedError] Error code: 529 ... 'message': 'Overloaded'`
    > 
- **Anthropic 로그 증거**: `claude-sonnet-4-6` 모델의 로그에 INPUT/OUTPUT TOKENS가 **'-'로 표시**된 것은, API 호출 시도는 있었으나 서버 과부하로 인해 요청이 처리되지 못하고 거부되었음을 나타냅니다.
- **결론**: 현재 사용 중인 Anthropic API 플랜의 한도 또는 일시적인 서버 이슈로 인해 `Sonnet 4.6` 호출이 실패하고 있습니다. 이는 `JSONDecodeError`와는 별개의 문제입니다.

### 추가 발견: 메타 패턴 추론 로직 오류

- **로그 증거**: Railway 로그에 `메타 패턴 DB 조회 실패` 오류가 꾸준히 기록되고 있습니다.
    
    > `메타 패턴 DB 조회 실패, 추론 건너뜀: Client error '400 Bad Request' for url '.../pattern_relations?select=...&relation_type=eq.inferred_by'`
    > 
- **분석**: `meta_pattern_inference.py`에서 `pattern_relations` 테이블을 조회하는 REST API URL에 문법 오류가 있는 것으로 보입니다. `inference_role` 컬럼명이 잘못되었거나 쿼리 파라미터 구성에 문제가 있을 가능성이 높습니다. 이로 인해 메타 패턴 추론 기능이 현재 전혀 작동하지 않고 있습니다.

---

### 해결 방안 제안

이 문제들을 해결하기 위해 다음과 같은 단계적 접근을 제안합니다.

### **1단계: `JSONDecodeError` 해결 (가장 시급)**

`pattern_matcher.py`의 `_robust_json_parse` 함수를 강화하여 `Sonnet 4.5`의 불안정한 출력을 처리해야 합니다. 기존 3단계 폴백(JSON 파싱 -> 재시도 -> 정규식)에 더해, **JSON 블록만 정확히 추출하는 전처리 로직을 추가**하는 것이 효과적입니다.

**개선된 `_robust_json_parse` 의사 코드:**

```python
def _robust_json_parse(text: str):
    # 1단계: 마크다운 JSON 블록 추출 (가장 안정적)
    # ```json ... ``` 부분을 먼저 찾아서 파싱 시도
    match = re.search(r"```json\\n(.*?)\\n```", text, re.DOTALL)
    if match:
        try:
            return json.loads(match.group(1))
        except json.JSONDecodeError:
            pass  # 실패 시 다음 단계로

    # 2단계: 일반 JSON 파싱 시도 (기존 로직)
    try:
        return json.loads(text)
    except json.JSONDecodeError:
        pass # 실패 시 다음 단계로

    # 3단계: 괄호 기반 JSON 객체 추출 (가장 너그러움)
    # '{'로 시작하고 '}'로 끝나는 가장 큰 덩어리를 찾음
    start = text.find('{')
    end = text.rfind('}')
    if start != -1 and end != -1:
        json_like_text = text[start:end+1]
        try:
            return json.loads(json_like_text)
        except json.JSONDecodeError:
            pass # 실패 시 다음 단계로

    # 4단계: 최후의 수단 - 정규식으로 패턴 코드만 추출
    pattern_codes = re.findall(r'["\\'](1-\\d-\\d)["\\']', text)
    if pattern_codes:
        # 정규식으로 추출된 코드를 기반으로 최소한의 JSON 구조를 복원
        return [{"pattern_code": code, "severity": "medium", "reasoning": "JSON parsing failed, extracted via regex."} for code in set(pattern_codes)]

    # 모든 시도 실패 시 예외 발생
    raise ValueError("Failed to parse or extract JSON from the model's output.")
```

### **2단계: `OverloadedError` 대응**

API 과부하는 코드 수정만으로 해결하기 어렵지만, 안정성을 높이기 위해 **지수 백오프(Exponential Backoff)를 적용한 재시도 로직**을 `anthropic` 클라이언트 호출 부분에 추가해야 합니다.

**적용 예시 (`pipeline.py` 또는 API 호출 래퍼):**

```python
import time
from anthropic import OverloadedError

# ...

for attempt in range(max_retries):
    try:
        # Sonnet 4.6 API 호출 로직
        report = report_generator.generate_report(...)
        return report
    except OverloadedError as e:
        if attempt < max_retries - 1:
            wait_time = (2 ** attempt)  # 1, 2, 4초...
            print(f"Overloaded. Retrying in {wait_time} seconds...")
            time.sleep(wait_time)
        else:
            raise e # 마지막 시도도 실패하면 에러 발생
```

### **3단계: 메타 패턴 추론 오류 수정**

`meta_pattern_inference.py`의 Supabase API 호출 URL을 검수하고 수정해야 합니다. `pattern_relations` 테이블의 정확한 컬럼명(`inference_rule` 또는 `role` 등)을 확인하고, `select` 쿼리 파라미터를 올바르게 수정해야 합니다.
