# CR-Check M6 Phase C — Phase Alpha 적용 후 사후 진단서

이전 세션에서 논의했던 핵심 아키텍처 개편안이 **CLI 프롬프트(phase_alpha_cli_prompt.md)를 거치는 과정에서 누락**되어, 근본적인 문제가 그대로 남아있거나 악화되었습니다. Test 디렉토리의 결과를 분석한 결과, 다음 3가지 핵심 실패 원인을 확인했습니다.

## 🚨 진단 결과 요약

1. **Bug A (벡터 검색 0건) 미해결**: 로깅만 추가되었을 뿐 검색 실패는 여전함.
2. **리포트 21KB 폭증 (가독성 파괴)**: CitationResolver의 각주형 분리 로직이 반영되지 않아 원문 전체가 본문에 삽입됨.
3. **API 한도 초과로 인한 타임아웃 4분 (Test 2, 3 실패)**: `max_tokens: 10000` 설정이 방치되어 에러 반환 및 4분간의 재시도 루프 발생.

## 🔍 상세 실패 원인 및 대안

### 1. 가독성 파괴 지속 (Citation Resolver 미수정)
- **증상**: Test 1(`diagnostic_20260404_234614.json`)은 리포트 생성이 성공했지만, 생성된 리포트 1-1 파일의 크기가 **21.8KB**에 달합니다. 
- **원인**: 이전 감리에서 "본문에는 규범 제목만 인용하고, 원문은 하단 부록(Footnote)으로 배치하자"고 강하게 승인/합의했으나, 이번 CLI 프롬프트에서 `citation_resolver.py` 수정 작업이 **아예 빠져 있었습니다.** 결과적으로 31개의 규범 원문이 본문 안에 그대로 치환되면서 리포트를 도저히 읽을 수 없는 상태가 되었습니다.
- **대안**: 즉시 `citation_resolver.py`를 수정하여 **하단 부록 형태(Footnote)**로 구조를 개편해야 합니다.

### 2. API 타임아웃 및 4분 이상 지연 (에러 핸들링 부재)
- **증상**: Test 2와 Test 3에서 모두 "리포트 생성 중 오류"로 fallback 되었으며, 소요 시간이 각각 266초, 256초입니다. 
- **원인**: 
  1. `report_generator.py`에 여전히 `max_tokens=10000`이 남아 있습니다. Claude 모델의 최대 출력 한도는 8192이므로, API가 오류를 반환했을 가능성이 큽니다.
  2. 에러가 발생해도 최대 재시도(지수 백오프)를 기다리며 4분 이상을 소진합니다.
  3. 특정 패턴(1-3-1, 1-1-1) 조합에서 규범 DB 매핑 연동 결과가 간헐적으로 0건을 반환할 때 방어 코드가 미흡합니다.
- **대안**: `max_tokens`를 **8192**로 하향 수정하고, 재시도 로직에서 API 권한/한도 에러일 경우는 즉각 중단(Fail-fast)하도록 수정해야 합니다.

### 3. Vector 검색 후보 항시 0건 (Bug A)
- **증상**: 3개 테스트 모두 `checkpoint_2_vector.candidate_count: 0`을 기록했습니다. 
- **원인**: CLI 프롬프트 지시사항이 "문제 해결"이 아니라 "문제 진단을 위한 로그 추가"에 그쳤기 때문에, Supabase Vector Similarity Search 결함(차원 불일치 또는 임계값 문제)이 계속해서 RAG 성능을 저하하고 있습니다.
- **대안**: 백엔드 터미널(알파 페이즈 수정으로 추가된 `search_vectors()` 로그)을 확인하여 임베딩 차원(dimension)이 일치하는지, `match_threshold` 값이 너무 높은지 즉시 확인하고 고쳐야 합니다.

## 🛠️ 권장하는 Next Steps (User 승인 요청)

CLI를 통하지 않고, 감리자인 저(Antigravity)가 직접 에디팅 툴을 사용해 위 문제를 수정하는 것을 제안합니다.

1. **`citation_resolver.py` 전면 수정**: 본문에는 `「신문윤리실천요강: 공정보도」` 수준의 짧은 제목만 배치하고 문서를 생성한 뒤, 끝에 `[참고 윤리 규범]` 리스트를 Append하는 방식으로 치환기를 개편합니다.
2. **`report_generator.py` 규격 수정**: `max_tokens=8192`로 제한을 맞추고, 불필요한 장기 타임아웃 방지책을 넣습니다.
3. **Vector 검색 로그 확인 플랜**: Vector 검색이 0건 호출되는 터미널 에러 로그의 원인만 파악해주시면, 해당 threshold/dimension 문제를 직권으로 수정하겠습니다.

이 계획대로 제가 직접 코드를 수정해도 될지 피드백 부탁드립니다.
