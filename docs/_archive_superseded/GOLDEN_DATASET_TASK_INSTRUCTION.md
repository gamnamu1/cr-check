## 0. 작업 목적

CR-Check Hybrid RAG 시스템의 평가 기준이 될 **골든 데이터셋 후보 목록**을 생성한다.
최종 골든 데이터셋은 20~30건이며, 이 작업에서는 **40~50건의 후보 풀**을 구조화하여 Gamnamu가 최종 선별할 수 있도록 준비한다.

**이 작업에서 Claude Code CLI가 하는 것:**
- 세 소스에서 후보 기사를 추출·정리
- 선별 기준에 따라 1차 분류 및 추천
- 구조화된 후보 목록 파일 생성

**이 작업에서 Claude Code CLI가 하지 않는 것:**
- 최종 선별 결정 (Gamnamu 전권)
- `expected_patterns` / `expected_ethics_codes` 레이블링 (Gamnamu + Claude 협업으로 별도 진행)
- 기사 본문 수집/저장 (저작권 원칙)

**파싱 시 주의사항:**

## 2. 작업 단계

### 작업 A: 심의자료 스캔 및 메타데이터 추출

**입력**: `yearly_data/` 내 4개 .md 파일
**출력**: 구조화된 심의 건 목록

각 심의 건에서 다음 필드를 추출한다:

| 필드 | 추출 방법 |
|------|-----------|
| `decision_no` | `**결정번호**:` 뒤의 값 |
| `decision_type` | `**결정유형**:` 뒤의 값 (주의/경고/시정권고 등) |
| `publisher` | `**언론사**:` 뒤의 값 |
| `title` | `# ` 뒤의 제목 |
| `year` | 파일명에서 추출 |
| `source_url` | `**원문 URL**:` 뒤의 값 |
| `applied_codes` | `## 적용 조항` 섹션의 전체 텍스트 |
| `reason_summary` | `## 이유` 섹션의 첫 3문장 (기사 핵심 문제 요약) |
| `cr_category_guess` | 아래 매핑 규칙으로 추정한 8대분류 |

**8대분류 추정 매핑 규칙** (적용 조항 키워드 기반):

```
적용 조항에 포함된 키워드 → 추정 대분류

"선거여론조사보도준칙"         → 1-1 진실성 (여론조사 오보)
"보도준칙" + "정확"            → 1-1 진실성
"취재원" / "출처"              → 1-2 투명성
"공정" / "균형" / "반론"       → 1-3 균형성
"광고" / "편집 독립"           → 1-4 독립성
"인권보도준칙"                 → 1-5 인권
"자살보도" / "자살예방"        → 1-5 인권 (자살보도 특수)
"재난보도준칙"                 → 1-6 전문성 (재난보도 특수)
"감염병보도준칙"               → 1-6 전문성
"선정" / "자극" / "제목"       → 1-7 언어
"혐오" / "차별"                → 1-7 언어 (혐오표현 특수)
"초상권" / "사생활"            → 1-5 인권
"명예훼손"                     → 1-5 인권

# 복수 매핑 가능 — 가장 핵심적인 것 하나를 primary로, 나머지는 secondary로 기록
```

**1차 필터링 기준 — 심의자료에서 우선 선별할 건:**
1. `decision_type`이 **'경고' 또는 '시정권고'**인 건을 우선 (패턴이 더 명확)
2. 같은 `applied_codes` 조합이 반복되면, **대표 1건만** 선택 (다양성 확보)
3. **연도별 분산**: 2021~2024 각 연도에서 최소 3건 이상
4. **적용 조항 다양성**: 최종 후보에서 가능한 한 많은 종류의 규범이 커버되도록

**목표 산출량**: 심의자료에서 **15~20건 후보**

### 작업 B: 문제 기사 파일명 스캔 및 분류

**입력**: `[문제 기사]/` 디렉토리의 파일명 목록 (104건+)
**출력**: 파일명 기반 분류 목록

각 파일에서 다음을 추출한다:

| 필드 | 추출 방법 |
|------|-----------|
| `filename` | 전체 파일명 |
| `date` | `[YYYY.MM.DD]` 부분 |
| `title` | 날짜와 언론사 사이의 제목 부분 |
| `publisher` | `_` 뒤, `.pdf` 앞의 언론사명 |
| `cr_category_guess` | 제목 키워드 기반 추정 (아래 참조) |

**제목 키워드 기반 8대분류 추정** (문제 기사용):

```
난민, 외국인 범죄, 이모님, 성소수자, 게이, 장애인    → 1-5 인권 / 1-7 언어(혐오)
여론조사, 지지율, 오차범위                            → 1-1 진실성
팩트체크                                              → 1-1 진실성 (단, 팩트체크 기사 자체의 문제)
"단독", "충격", "일파만파", "벼락", 낚시성 느낌       → 1-7 언어
파업, 노조, 임금, 노동 + 편향적 프레이밍 느낌         → 1-3 균형성 / 1-4 독립성
특정 정치인 공세/옹호 일변도                          → 1-3 균형성
탈원전, 포퓰리즘 등 가치판단 포함 제목                → 1-3 균형성
부동산, 집값 + 자극적 프레이밍                        → 1-7 언어 / 1-1 진실성
```

**1차 필터링 기준 — 문제 기사에서 우선 선별할 건:**
1. **심의자료가 커버하지 못하는 대분류**를 우선 채움:
   - 1-4 독립성 (외부 압력, 상업적 동기)
   - 1-5 인권 (혐오, 차별, 난민, 성소수자, 장애인)
   - 1-7 언어 (낚시성 제목, 자극적 표현)
   - 1-8 디지털 (뉴스 어뷰징)
2. **복합 패턴이 예상되는 기사**를 우선 (제목에서 여러 문제가 겹쳐 보이는 것)
3. **최근 기사 우선**: 2024~2026년 기사가 URL 접근성이 높아 골든 데이터셋 활용에 유리

**목표 산출량**: 문제 기사에서 **15~20건 후보**

### 작업 C: 이달의 기자상 수상작 목록 수집

**입력**: `journalist_award_links.csv`의 URL 목록
**작업**: 2021~2025년 범위의 수상작 페이지에서 개별 수상작 정보를 수집

**수집 범위**: 2021~2025년 (60개월분, 약 60개 URL)
- 전부 수집할 필요 없음. **연도별 3~4개 페이지를 샘플링**하여 총 15~20개 페이지 수집
- 각 페이지에서 수상작 제목, 언론사, 수상 부문(취재보도/기획보도/탐사보도 등)을 추출

**수집 시 유의사항:**
- journalist.or.kr 페이지의 HTML 구조를 확인하고, 수상작 목록이 어떤 형태로 게시되는지 파악
- 페이지 접근이 안 되면 무리하지 말고, 접근 가능한 페이지만 수집
- 수상작 제목만으로도 기사 주제를 파악할 수 있으므로, 기사 원문 URL까지 추적할 필요는 없음

**True Negative 후보 선정 기준:**

수상작 중에서 "시스템이 오탐할 가능성이 높은" 기사를 우선 선별한다:
- 탐사보도/심층보도 부문 수상작 중 **익명 취재원을 다수 사용했을 가능성이 높은 것** (내부고발, 비리 추적 등) → 1-1-1(불분명한 출처) 오탐 테스트
- **선정적으로 보일 수 있는 제목의 심층 보도** → 1-7-3(낚시성 제목) 오탐 테스트
- **한쪽 입장 중심이지만 피해자 보호 목적인 인권 보도** → 1-3-1(관점 다양성 부족) 오탐 테스트
- 단순히 "평범하게 잘 쓴 스트레이트 기사"는 True Negative으로서 가치가 낮으므로 피할 것

**목표 산출량**: 수상작에서 **5~8건 True Negative 후보** (Gamnamu가 2~3건 최종 선택)

## 4. 전체 선별 원칙 (밸런스 체크)

Claude Code CLI는 후보 목록 작성 완료 후, 다음 기준으로 밸런스를 자가 점검한다:

### 4-1. 8대분류 커버리지

| 대분류 | 최소 후보 수 | 비고 |
|--------|-------------|------|
| 1-1 진실성 | 5건+ | 심의자료에 풍부 |
| 1-2 투명성 | 3건+ | 심의자료 + 문제 기사 |
| 1-3 균형성 | 5건+ | 양쪽 소스에 풍부 |
| 1-4 독립성 | 3건+ | 문제 기사에서 주로 확보 |
| 1-5 인권 | 5건+ | 문제 기사에서 주로 확보 |
| 1-6 전문성 | 3건+ | 심의자료 (재난/감염병 보도) |
| 1-7 언어 | 4건+ | 문제 기사에서 주로 확보 |
| 1-8 디지털 | 2건+ | 확보 어려울 수 있음 — 없으면 명시 |
| True Negative | 5건+ | 수상작에서 확보 |

### 4-2. 난이도 분포

| 난이도 | 비율 | 설명 |
|--------|------|------|
| easy | ~40% | 패턴이 한눈에 보이는 기사, 적용 조항 명확 |
| medium | ~40% | 복합 패턴, 여러 대분류 겹침 |
| hard | ~20% | 메타 패턴 추론 필요, True Negative |

### 4-3. 복합 패턴 기사

전체 후보 중 **최소 10건 이상**은 `is_complex_pattern: true`여야 한다.
(실제 뉴스 분석에서 가장 흔한 시나리오가 복합 패턴이므로)

### 4-4. 연도 분산

심의자료 + 문제 기사 통합 기준으로 특정 연도에 60% 이상 집중되지 않도록 한다.

## 5. 실행 시 유의사항

### 5-1. 심의자료 파싱

- 파일이 매우 길다 (15,000~23,000줄). 전체를 한번에 읽으려 하지 말고, **청크 단위로 읽으면서** 각 건의 메타데이터를 추출하라.
- `--- (원본 파일:` 구분자를 기준으로 건을 분리한다.
- `## 적용 조항` 이후, 다음 `---` 구분자까지가 해당 건의 적용 조항 텍스트이다.
- 일부 건은 `## 적용 조항`이 없을 수 있다 — 그런 건은 건너뛴다.

### 5-2. 문제 기사 분류

- **PDF 본문을 읽지 않는다.** 파일명만으로 분류한다.
- 제목만으로 8대분류를 확정하기 어려운 경우, `cr_category_guess`를 `"uncertain"`으로 두고 `selection_reason`에 추정 근거를 기술한다.
- 파일명의 언론사가 잘려있거나 오타가 있을 수 있다 (예: `문화읿보` → `문화일보`). 가능하면 보정한다.

### 5-3. 수상작 웹 수집

- `journalist.or.kr` 페이지 접근 시 **속도 제한에 유의**한다. 페이지 간 1~2초 간격을 둔다.
- 페이지 접근이 실패하면 해당 월을 건너뛰고 다음으로 넘어간다. 실패 목록을 별도 기록한다.
- HTML 구조가 예상과 다르면, 가능한 범위까지만 수집하고 나머지는 `수동 확인 필요`로 표시한다.
- 수상작이 연재/시리즈인 경우, 대표 1건만 기록한다.

### 5-4. 정치적 중립성

- 후보 선별 시 **특정 정당이나 정치 성향에 치우치지 않도록** 한다.
- 문제 기사 중 정치 관련 기사가 다수인데, 여야 양쪽에 대한 편향 보도가 균형 있게 포함되도록 한다.
- 이는 골든 데이터셋의 공정성과 CR-Check 시스템의 신뢰성을 위해 필수적이다.

### 5-5. 저작권 원칙

- **기사 본문을 저장하지 않는다.** URL, 제목, 메타데이터만 기록한다.
- 심의자료의 `reason_summary`는 신문윤리위원회의 공개 심의 결정문에서 발췌하는 것이므로 참조 용도로 허용된다.
