# 골든 데이터셋 — 실제 기사 역추적 리서치 브리프

> **작성**: 2026-03-23
> **목적**: 외부 리서치에서 수집된 "메타 기사"(비평·해설)에서 언급된 **실제 문제 기사**를 역추적하여 골든 데이터셋 후보로 확보
> **실행 대상**: Perplexity, Gemini, 또는 수동 검색

---

## 0. 이 브리프가 필요한 이유

외부 리서처들이 수집한 자료 중 상당수가 "어뷰징 기사 그 자체"가 아니라
"어뷰징에 대한 미디어 비평 기사"였다.

CR-Check이 분석할 골든 데이터셋에 필요한 것은 **문제 기사 그 자체**이다.
그러나 메타 기사 안에는 실제 문제 기사의 제목·언론사·시기가 언급되어 있어,
이를 단서로 역추적하면 1-8 디지털 영역의 후보를 확보할 수 있다.

**핵심 원칙**: 비평 기사가 아니라, 비평 기사가 비판하는 **원본 기사**를 찾는 것이다.

---

## 1. 리드 A: '재조명' 어뷰징 실제 기사

**단서 출처**: Perplexity1 A-1-2 (오마이뉴스, 2020-07)
**메타 기사 URL**: https://star.ohmynews.com/NWS_Web/OhmyStar/at_pg_m.aspx?CNTN_CD=A0002657609

**찾아야 할 것**:
- 이 기사 안에서 비판하는 **실제 '재조명' 어뷰징 기사 3~5건**의 제목, 언론사, URL
- 예: "○○ 재조명됐다" 식의 제목으로, 원본 기사와 거의 동일한 내용을 반복한 기사
- **검색 쿼리 제안**: `site:naver.com "재조명" 2020` + 동일 이슈 기사 5건 이상 쏟아진 사례

**기대 산출물**: 동일 이슈에 대해 제목만 바꿔 반복 전송된 기사 묶음 2~3세트
**CR-Check 분류**: 1-8 디지털 (뉴스 어뷰징/기사 재활용)

## 2. 리드 B: 조선일보 AI 어시스턴트로 작성된 실제 기사

**단서 출처**: Perplexity1 A-3-1 (기자협회보, 2024-01)
**메타 기사 URL**: http://journalist.or.kr/m/m_article.html?no=55009

**찾아야 할 것**:
- 조선일보의 'AI 기사 작성 어시스턴트'로 작성된 것으로 추정되는 **실제 기사** 1~2건
- AI 생성 여부가 표기되지 않은 기사가 있다면 특히 가치 있음
- 기자협회보 원문에 구체적 기사 제목·날짜가 언급되어 있을 가능성

**검색 쿼리 제안**: 
- `조선일보 AI 기사 생성 2024`
- 기자협회보 원문을 fetch하여 구체 사례 추출

**기대 산출물**: AI 생성(또는 AI 보조) 기사로 확인/추정되는 실제 기사 1~2건 + URL
**CR-Check 분류**: 1-8 디지털 (AI 생성 기사 투명성)

**CR-Check 분류**: 1-8 디지털 (기사 재활용) + 1-5 인권 (2차 가해)

## 9. 산출물 형식

각 리드에서 실제 기사를 찾으면, 다음 형식으로 정리한다:

```json
{
  "candidate_id": "T-XX",
  "source": "역추적-리드X",
  "title": "[실제 기사 제목]",
  "publisher": "[언론사]",
  "date": "YYYY-MM-DD",
  "decision_no": "[있으면 기입]",
  "source_url": "[기사 URL]",
  "cr_category_primary": "1-8",
  "lead_source_url": "[단서가 된 메타 기사 URL]",
  "tracking_note": "[어떻게 추적했는지 간략 기록]"
}
```
