# 골든 데이터셋 선별 원칙 — 방송/영상 콘텐츠 취급 기준

> **확정일**: 2026-03-23
> **확정자**: Gamnamu
> **적용 범위**: 전체 골든 데이터셋 후보 풀 (1차·2차·외부 라운드 전체)

---

## 원칙: CR-Check 파이프라인 입력 적합성 기준

골든 데이터셋의 모든 후보는 **CR-Check이 실제로 분석할 수 있는 형태의 입력**이어야 한다.

CR-Check의 분석 파이프라인은 다음을 전제한다:
- 사용자가 **기사 URL 또는 텍스트**를 입력
- Haiku가 **텍스트 기반** 패턴을 스크리닝
- Sonnet이 **텍스트 기반** 규범 매핑으로 상세 분석
- 394개 `ethics_codes_mapping.json` 엔트리는 **인쇄/온라인 매체의 텍스트 보도**를 전제로 설계

따라서 **텍스트 기사 형태로 CR-Check에 입력할 수 없는 콘텐츠**는 골든 데이터셋에서 제외한다.

---

## 제외 기준

다음 중 **하나라도 해당**하면 골든 데이터셋에서 제외한다:

1. **방송 리포트 중심 콘텐츠**: TV/라디오 방송 리포트로서, 앵커 멘트 + 기자 리포트 + 인터뷰 사운드바이트 + 내레이션이 결합된 형식. 텍스트 변환 시 원래 방송 문맥(영상, 톤, 편집 순서)이 소실됨.

2. **수상 부문이 "방송부문"인 True Negative 후보**: 방송 리포트로서 수상한 것이므로, 웹 텍스트 축약본으로 TN 가치를 평가할 수 없음.

3. **분석의 핵심이 영상·이미지·음성에 있는 사례**: 텍스트 기사가 존재하더라도, 문제(또는 우수성)의 핵심이 비텍스트 요소에 있는 경우.

4. **유튜브 영상 전용 콘텐츠, 팟캐스트**: 텍스트 기사 형태가 존재하지 않는 경우.

---

## 유지 기준

방송사 소속이라도 다음에 해당하면 **유지**한다:

1. **웹에 충분한 분량의 독립 텍스트 기사가 존재**: KBS·MBC·JTBC 등 방송사의 웹 뉴스 섹션에 텍스트 기사로 게재된 경우. 이 텍스트 기사 자체를 CR-Check에 입력하여 분석 가능.

2. **수상 부문이 "신문/통신부문"인 경우**: 텍스트 기반 저널리즘으로서 수상한 것.

3. **텍스트 탐사보도 기사를 병행 게재하는 매체**: 뉴스타파처럼 영상과 텍스트를 동시에 제작하는 경우, 텍스트 기사가 독립적으로 완결성을 갖추면 유지.
