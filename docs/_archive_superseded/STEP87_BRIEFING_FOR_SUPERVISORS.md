# CR-Check STEP 87 리포트 품질 개선 — 작업 완료 브리핑 (Manus용)

## 요약

E2E 테스트에서 확인된 평가 리포트 품질 저하 문제를 Claude.ai(설계·감리) + Claude Code CLI(구현) 체계로 해결했습니다. 이전에 여러 감리자분들이 진단해주신 내용을 종합 분석한 후, 3단계(Phase α → β → 임베딩 생성)에 걸쳐 수정을 진행하여 전체 파이프라인이 정상 작동하고, 리포트 품질이 기존 시스템 수준으로 회복되었습니다.

---

## 해결된 문제들

### Phase α — 버그 수정 3건

**Bug A (벡터 검색 항상 0건):**
- 원인: `patterns` 테이블의 `description_embedding` 컬럼에 벡터 데이터가 한 번도 삽입되지 않았음
- 해결: `scripts/generate_embeddings.py` 실행 → 28개 소분류 패턴 + 373개 규범에 임베딩 생성 완료 (1536차원, text-embedding-3-small)
- 결과: 벡터 검색이 정상 작동 (테스트에서 candidate_count: 11건 반환 확인)

**Bug B (Sonnet Solo JSON 파싱 전체 실패):**
- 원인: Sonnet Solo가 `matched_text` 필드에 복수 문자열을 쉼표로 나열하여 유효하지 않은 JSON 생성 → `json.loads()` 실패 → 전체 탐지 결과 유실
- 해결: `_parse_solo_response()`에 3단계 fallback 구현 (정상 파싱 → JSON 복구 → 정규식 추출) + 프롬프트에 JSON 형식 경고 추가
- 결과: 깨진 JSON에서도 패턴 코드를 복구하여 탐지 결과 보존

**Bug C (규범 조회 간헐적 0건):**
- 원인: `get_ethics_for_patterns` RPC가 특정 패턴 ID 조합에서 간헐적으로 0건 반환 (재귀 CTE + 필터 조건의 간헐적 문제로 추정)
- 해결: `fetch_ethics_for_patterns()`에 재시도 로직 + REST API 직접 JOIN 조회 fallback 추가
- 결과: RPC 실패 시 REST API가 자동 우회하여 규범 안정적 조회

### Phase β — 인용 구조 전환 (핵심 변경)

**근본 원인으로 확인된 "인용 대체 아키텍처" 전면 전환:**

5개 감리 진단서를 종합한 결과, **Sonnet이 규범 원문 없이 리포트를 작성하고 `<cite>` 태그를 기계적으로 후치환하는 구조**가 품질 저하의 근본 원인으로 확인되었습니다.

- `_SONNET_SYSTEM_PROMPT` 전면 교체: cite 태그 지시 삭제 → "규범 원문을 읽고 자연스럽게 녹여 쓰세요" + 조항 번호 명시 + 핵심 문구 발췌 인용 방식으로 변경
- `citation_resolver.py` 비활성화: `pipeline.py`에서 호출 루프를 주석 처리 (코드 자체는 보존)
- `_build_ethics_context()` 형식 변경: Sonnet이 조항 번호를 쉽게 참조할 수 있는 구조로
- 프롬프트 정비: 롤업 지시 정정, 3종 톤 재설계, 서술 스타일 가이드 추가 ("문제" 반복 금지, 중간제목 제한 등)
