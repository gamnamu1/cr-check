def normalize_url(url: str) -> str:
    """URL에서 트래킹 파라미터를 제거하여 캐시 키로 사용."""
    from urllib.parse import urlparse, urlunparse, parse_qs, urlencode
    parsed = urlparse(url)
    # 제거할 파라미터 접두어
    remove_prefixes = ('utm_', 'fbclid', 'gclid', 'mc_', 'ref', 'source')
    params = parse_qs(parsed.query)
    cleaned = {k: v for k, v in params.items()
               if not any(k.startswith(p) for p in remove_prefixes)}
    clean_query = urlencode(cleaned, doseq=True)
    return urlunparse(parsed._replace(query=clean_query, fragment=''))

### 이 문서의 위치

이 문서는 SESSION_CONTEXT v22를 기반으로, **로컬 코드베이스를 전수 확인한 뒤** 작성되고,
**4인 감리(안티그래비티, 마누스, 제미나이, 퍼플렉시티) 피드백 6건이 반영된** Phase D 최종 실행 계획이다.
이전 버전(`Phase_D_Execution_Plan.md`)보다 이 문서가 우선한다.

### 핵심 맥락

1. **Phase D + F-2가 통합되어 있다.** 이것은 의도적 결정이다. 스키마를 1회 설계로 완결하고, 로컬에서 전체 흐름(저장→조회→렌더링)을 완성한 뒤 Phase E에서 일괄 배포한다.
2. **캐시 우선 정책이 확정되었다.** 같은 URL 재분석 시 DB에서 기존 결과를 즉시 반환한다.
3. **★ URL 정규화가 포함되었다 (감리 반영).** utm 등 트래킹 파라미터를 제거하여 캐시 효율을 높인다.
4. **프론트엔드는 App Router다.** `frontend/app/` 구조 확인 완료. Pages Router 관련 코드를 생성하지 말 것.
5. **AnalyzeResponse의 article_info 타입이 `Dict[str, str]`에서 `Dict[str, Any]`로 변경되어야 한다.** share_id(str), is_cached(bool) 등 비-string 필드가 추가되기 때문.
6. **★ UPSERT 시 `?on_conflict=url` 파라미터를 반드시 명시할 것 (감리 반영).** 없으면 409 에러 발생.
7. **진단 JSON은 DB에 넣지 않는다.** `backend/diagnostics/`에 파일로 유지.
8. **share_id 충돌 재시도 로직(3회)을 반드시 포함할 것.**
9. **★ 기존 result 페이지에서도 캐시 안내 배너를 표시할 것 (감리 반영).** CachedBanner 컴포넌트를 두 페이지에서 공유.
10. **v22까지의 교훈 1~27 모두 유효.**
