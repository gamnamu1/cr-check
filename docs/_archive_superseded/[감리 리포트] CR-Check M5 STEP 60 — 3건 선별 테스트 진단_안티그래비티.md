### 📋 [독립 감리 리포트] CR-Check M5 STEP 60 — 3건 선별 테스트 진단

[Claude.ai](http://claude.ai/) 파트너의 진단과 옵션들을 비Anthropic 관점(LLM의 작동 원리 및 시스템 프롬프팅 최적화)에서 독립적으로 검토한 결과입니다.

### 1. 핵심 모순에 대한 독립적 원인 진단

현재 발생한 모순(B2-10 허위면제 vs E-19 오탐)의 진짜 원인은 **"LLM에게 '생각(CoT)'할 텍스트 공간을 주지 않고, 머릿속에서 Binary Gate(1단계)를 통과하라고 강요했기 때문"**입니다.

- 프롬프트의 *"아래 2단계를 진행하지 말고 빈 배열 []을 즉시 반환하세요"*라는 지시문은 LLM 입장에서 **Zero-shot** 판단입니다.
- 아무런 추측 과정(reasoning)을 텍스트로 내뱉지 않은 상태에서 기사만 보고 직관적으로 `[]`를 결정해 버리기 때문에, B2-10처럼 겉보기에 멀쩡한 '팩트체크' 타이틀을 달고 있으면 TN 강화 문구에 홀려 무턱대고 `[]`를 반환합니다.
- 반대로 E-19는 '국가 비상금', '대통령실' 등 갈등 어휘가 가득하므로 0.1초 만에 "이건 문제가 있다"고 직관 발동 후 2단계로 넘어가 기어이 오탐 패턴(1-7-6)을 쥐어짜낸 것입니다.

### 2. Claude.ai의 옵션 A/B/C 평가

- ❌ **A안 (1단계를 신뢰도 스코어로 전환)**: **반대**. 숫자를 매기는 행위 역시 이유를 쓰지 않고 숫자만 뱉는다면 Zero-shot 모순을 해결하지 못하며, 프롬프트 복잡도만 올립니다.
- ⚠️ **B안 (TN 강화 문구를 few-shot 내부에만 배치)**: **부분 찬성**. 과잉 보호(B2-10)의 부작용은 약간 줄이겠지만, E-19를 보호하지 못하는 문제는 해결할 수 없는 미봉책입니다.
- ✅ **C안 (1단계 완전 제거, 2단계에서 통합 판별)**: **강력 찬성**. LLM은 배열`[]`을 출력하기 시작하면 뒤로 돌아갈 수 없습니다. 어설픈 프리패스(Gate) 제도를 없애고, "이 패턴이 정당한 탐사보도의 범주에 속하는가?"를 패턴별 `reasoning` 안에서 논증하게 만들어야 합니다.

### 3. 비Anthropic 관점의 추가 아이디어 (강제 CoT 스키마 도입)

가장 근본적인 해결책은 **JSON 스키마 자체를 래핑(Wrapping)하여 전체 총평을 먼저 쓰게 강제**하는 것입니다. (Gemini, OpenAI 등에서 가장 효과명확한 구조)
*현 `parse_haiku_response` 코드가 배열 파싱만 고집하므로 당장 적용이 번거로울 수 있으나, 정규식 파서에 여유가 있다면 프롬프트를 아래처럼 바꾸는 것이 "실버 불릿"입니다.*

```json
{
  "overall_assessment": "이 기사는 구체적 문서를 근거로 한 정상적 탐사보도이므로 윤리규범 위반이 아니다...",
  "detections": []  // 위에서 스스로 쓴 총평 덕분에 빈 배열을 쉽게 도출
}
```

- 위 스키마 변경이 당장 부담스럽다면 **C안에 전적으로 동의**하며, 프롬프트의 `reasoning` 룰에 *"왜 이것이 탐사보도/당연한 비판이 아닌 윤리 위반인지 논증하라"*고 지시하는 것으로 우회해야 합니다.

### 4. E-15 문제 (벡터 검색 CR=0.00) 단기 대응 의견

코드 레벨의 프롬프트나 청킹([chunker.py](http://chunker.py/))을 건드리지 않고 할 수 있는 가장 강력한 단기 해결책은 **데이터 임플란트(Data Implant)**입니다.

- **원인**: E-15 기사 내용("정치인 따옴표 막말")과 DB에 들어있는 1-7-2, 1-7-5 윤리규범 원문의 어휘(Vector Scope)가 겹치지 않아 검색이 빗나갑니다.
- **대응책 (DB 업데이트)**: Supabase의 `patterns` 테이블에서 1-7-5(혐오 표현)나 1-7-2(제목 과장)의 `description` 컬럼 텍스트 맨 끝에 **"따옴표 저널리즘", "막말 그대로 인용", "논란 발언 여과 없이 보도"** 등의 키워드를 UPDATE 쳐서 밀어 넣습니다.
- 이렇게 "패턴 설명의 밀도"를 보강해주면 다음 벤치마크부터 코드 수정 1줄 없이도 귀신같이 벡터 후보망(Candidate Recall)에 낚여 올라오게 됩니다.
